
# Contributing to Syntra Lite

Thank you for considering contributing to **Syntra Lite**! Your contributions help us improve and provide better functionality. Please follow the guidelines below to ensure a smooth and efficient workflow.

## Commit Message Guidelines

When making commits, please use the following prefixes:

* **Feature Commits**
  `feat:` – for new features
  Example: `feat: add new game loader`

* **Fix Commits**
  `fix:` – for bug fixes
  Example: `fix: correct iframe sandbox config`

* **Refactor Commits**
  `refactor:` – for code improvements that do not change behavior
  Example: `refactor: simplify routing logic`

* **Chore Commits**
  `chore:` – for maintenance or routine changes
  Example: `chore: remove outdated assets`

* **Additions**
  `add:` – when adding new modules, functions, or enhancements
  Example: `add: preload script for ui`

* **Updates**
  `update:` – modifying existing functionality or improving performance
  Example: `update: optimize game loading times`

* **Version Changes**
  Version format: `1.x.x`
  Example: `1.6.7 - Added: new import system`

---

## Pull Request Guidelines

1. **Fork the Repository**
   Create your own fork of **Syntra Lite**.

2. **Create a Branch**
   Use a descriptive branch name that reflects your work.

3. **Make Your Changes**
   Follow the commit message rules above.

4. **Test Your Changes**
   Ensure your contribution works as expected.

5. **Push Your Branch**
   Push the branch to your fork.

6. **Submit a Pull Request**
   Explain what you changed and why it’s needed.
